바로 구현보다, 설계 후 구현이 훨씬 잘 먹혔던 이유입니다.

35

바로 구현

  • 방향이 자주 흔들린다.
  • 구조가 남지 않는다.
  • 수정 시 전체가 다시 흔들린다.

설계 후 구현

  • 이미 정리된 구조를 더 나은 구조로 재배치한다.
  • 구현을 역할 단위로 분리하기 쉽다.
  • 사람이 유지보수 가능한 형태로 수렴한다.

AI는 코드를 더 많이 쓰는 도구보다, 구조를 더 잘 다시 놓게 만드는 도구에 가까웠습니다.
