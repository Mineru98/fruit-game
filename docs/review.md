# 전체적인 리뷰

이 문서는 실제 렌더 기준 슬라이드 58장을 전부 검토한 통합 리뷰입니다.
검토 기준은 `http://127.0.0.1:4173/decks/vibe-coding-2026/viewer.html` 뷰어, 각 슬라이드 HTML, 슬라이드별 PNG 캡처입니다.

## 우선 수정 항목

- 전반적으로 글이 도중에 잘리거나, 좁은 박스에서 낱말 단위 줄바꿈이 나는 장면이 많습니다. 발표에서는 이런 끊김이 청중 피로를 크게 만듭니다. 전 범위에서 먼저 정리해야 합니다.
- 제목이 긴 슬라이드가 연속되는 구간이 많아 상단 피로가 누적됩니다. 장표 제목은 더 짧고 강하게 압축하는 편이 좋습니다.
- 도식과 캡처 이미지가 많은데, 실제 발표 거리에서는 내부 라벨이 읽히지 않는 경우가 반복됩니다. 작게 많이 보여주기보다 핵심 일부를 크게 보여주는 편이 낫습니다.
- 텍스트 카드만 나란히 놓인 슬라이드가 중간중간 반복되어 리듬이 평평해집니다. 사례 화면, 확대 이미지, 흐름 도식, 전환 장표를 더 의도적으로 섞어야 합니다.
- 여러 장에서 하단 여백이 과하게 남거나, 반대로 좌상단에만 정보가 몰리는 문제가 있습니다. 시선 중심을 다시 잡아야 합니다.

## 기존 지적 사항 반영 체크

### 1 페이지
- `Playwright`는 역할 중심 표현인 `테스트 자동화` 또는 `검증 자동화`로 바꾸는 편이 좋습니다.
- `40-minute map` 영역은 제거하는 편이 낫습니다.
- `톤앤매너 · Dark blueprint, neon HUD, Pretendard`는 청중용 정보가 아니라 삭제하는 것이 맞습니다.

### 3 페이지
- 현재도 좌우 분산이 커서 2:1 상하 구조로 바꾸는 편이 더 적절합니다.
- `vibe-signal.svg` 계열 그래프는 가장자리 잘림과 작은 라벨 문제가 남아 있습니다.
- 단조로운 게이지형 애니메이션은 줄이고, 정보 전달이 우선이 되도록 재구성해야 합니다.

### 4 페이지
- 트윗 캡처는 있지만 안드레 카파시라는 인물 맥락은 여전히 약합니다.
- 카파시 사진이나 프로필 컷을 별도 장 또는 보조 시각 요소로 추가하는 편이 좋습니다.

### 12 페이지
- 현재 1x6 구조의 한계가 큽니다.
- 2x3 구조로 재배치하고, 연결선도 실제로 잘 읽히는 방식으로 다시 설계해야 합니다.

## 범위별 핵심 관찰

### 01~15
- 01페이지는 기존 피드백이 충분히 반영되지 않았습니다. `40-minute map`, `톤앤매너`, 과한 키워드 칩 때문에 오프닝 집중도가 아직 낮습니다.
- 03페이지는 여전히 좌우 분산이 크고, 우측 그래프 라벨 잘림과 작은 파트 카드 때문에 가독성이 떨어집니다.
- 04페이지는 트윗 캡처는 들어갔지만, 요청된 안드레 카파시 인물 맥락은 여전히 약해서 감정적 접점이 부족합니다.
- 12페이지는 기존 지적대로 1x6 구조가 가장 큰 문제입니다. 2x3 재배치와 더 명확한 연결선 설계가 필요합니다.
- 14페이지는 다이어그램 자체는 의미 있지만 실제 발표 화면에서 너무 작아 “보여주기”보다 “있어 보이기”에 가까운 상태입니다.

### 16~30
- 17, 19, 22, 26, 29는 제목이 길어서 상단 피로도가 높고 첫 인상이 무겁습니다.
- 18, 22, 29는 도식 자체의 방향은 좋지만 실제 발표 거리에서 내부 라벨이 잘 안 읽힙니다.
- 21, 23, 24, 30은 텍스트 비중이 높고 시각적 증거가 약해 설명 슬라이드처럼 평평하게 느껴집니다.
- 24는 좌상단 편중이 가장 크고, 16, 17, 26도 여백 대비 텍스트 블록 위치가 치우쳐 화면 밸런스가 불안정합니다.
- 28, 30은 사례 전환의 잠재력은 큰데 영상/썸네일 증거가 약하거나 브라우저 UI가 섞여 있어 발표용 완성도가 떨어집니다.

### 31~44
- 31, 33, 34, 43은 핵심 도식이나 결과 이미지가 작아서 “근거 슬라이드”가 아니라 “썸네일 슬라이드”처럼 보입니다.
- 35, 38, 40, 41, 42는 제목 길이가 길어 상단 호흡이 무겁고, 연속 구간 피로가 누적됩니다.
- 32, 39, 42, 44는 텍스트 카드 중심이라 화면 밀도가 낮고 시각 증거가 부족합니다.
- 31과 36은 줄바꿈과 레이아웃 폭 문제 때문에 문장이 자연스럽게 읽히지 않고, 발표 리듬이 살짝 끊깁니다.
- 37, 41, 44는 메시지는 좋은데 위계가 평평해서 어디를 먼저 봐야 하는지 한 번에 꽂히지 않습니다.

### 45~58
- 후반부 전체에서 제목 길이가 길어지면서 호흡이 무거워지고, 특히 45, 46, 52에서 첫 인상 피로가 큽니다.
- `harness-blueprint.svg` 계열 구조도가 50, 52, 54에서 반복되어 시각 리듬이 단조로워집니다.
- 53은 6단계 프로세스를 가로 1열로 늘여서 텍스트가 작고 하단 여백이 과하게 남습니다.
- 47, 48은 소재는 좋지만 이미지와 숫자 대비 핵심 메시지 위계가 약해서 발표 임팩트가 덜 살아납니다.
- 56, 57, 58 마무리 구간은 메시지는 선명하지만 대표 문장 하나를 더 강하게 세우면 엔딩 기억점이 훨씬 좋아집니다.

# 슬라이드별 상세 리뷰

## 슬라이드 01
### 좋은 점
- 커버의 톤이 전체적으로 안정적이고, 어두운 배경과 청록 포인트가 발표 주제와 잘 맞습니다.
- 좌측 제목, 우측 핵심 메시지, 키워드 패널의 3분할은 첫 장으로서 구조 이해가 빠릅니다.
- 제목 크기와 본문 크기의 대비가 분명해서 첫 인상은 강합니다.

### 문제
- `40 MIN TECH TALK`, `톤앤매너`, `40-minute map`은 청중용 메시지라기보다 내부 메모처럼 보여서 첫 장의 집중도를 떨어뜨립니다.
- `Playwright` 키워드는 발표의 핵심 개념보다 도구명을 먼저 노출해 표지의 관점이 좁아집니다.
- 우측 하단 막대 차트는 의미를 설명하지 못해 장식 이상의 역할을 하지 못합니다.

### 개선 제안
- 표지에서는 메타 정보와 도구명을 줄이고, 주제와 핵심 주장만 남겨 한 장의 선언처럼 보이게 하세요.
- `Playwright`는 `테스트 자동화`나 `검증 자동화`처럼 역할 중심 문구로 바꾸는 편이 좋습니다.
- `40-minute map`은 삭제하거나, 발표 핵심 흐름 한 줄로 치환하면 화면 밀도가 더 좋아집니다.

## 슬라이드 02
### 좋은 점
- 중앙 선언문이 강해서 발표의 첫 논지를 바로 꽂아 주는 힘이 있습니다.
- 상단 설명과 중앙 카피의 역할이 분리되어 있어 메시지 구조는 읽기 쉽습니다.
- 전 장보다 시각적 소음이 줄어들어 숨 고르는 구간으로 기능합니다.

### 문제
- 상단 설명, 부제, 중앙 문장이 같은 뜻을 반복해 정보 효율이 낮습니다.
- 중앙 문장이 화면 한가운데만 떠 있어 여백은 좋지만 변주가 약합니다.
- 강조색이 두 군데 쓰이지만 역할 차이가 약해서 색의 의미가 충분히 분리되지 않습니다.

### 개선 제안
- 부제를 더 짧게 줄이고, 중앙 문장을 더 강한 선언문으로 키우세요.
- 색상은 `문서화`와 `Harness Engineering`의 역할 차이를 의도적으로 나누는 편이 좋습니다.
- 중앙 아래에 짧은 보조 근거나 작은 시각 요소를 넣어 시선이 머무를 이유를 만들어 주세요.

## 슬라이드 03
### 좋은 점
- 5단계 아젠다가 한눈에 보여서 발표 흐름 예고에는 적합합니다.
- 오른쪽 그래프형 비주얼이 정적 텍스트 슬라이드의 단조로움을 조금 풀어 줍니다.
- 단계 번호와 이름이 함께 있어 청중이 전체 구조를 빠르게 기억할 수 있습니다.

### 문제
- 기존 리뷰에서 요청한 상하 2:1 전환이 반영되지 않아, 좌우가 서로 경쟁합니다.
- 왼쪽 카드 폭이 좁아 줄바꿈이 잦고, 오른쪽 그래프의 끝 라벨은 가장자리에 붙어 답답합니다.
- `PART MAP` 막대와 하단 진행 바가 모두 비슷한 역할을 하면서 시선을 분산시킵니다.

### 개선 제안
- 2:1 상하 구조로 바꿔 위에는 흐름, 아래에는 신호 그래프를 두는 편이 더 읽기 좋습니다.
- 오른쪽 그래프는 텍스트를 안전 영역 안쪽으로 밀고, 필요하면 한 단계 확대하세요.
- 진행 상태는 막대 여러 개보다 단일 하이라이트로 단순화하는 편이 더 기능적입니다.

## 슬라이드 04
### 좋은 점
- 왼쪽 설명과 오른쪽 트윗 캡처가 분리되어 있어 계기와 근거를 붙여 보여주기 좋습니다.
- 실제 트윗을 크게 보여줘서 이야기의 출발점이 구체적으로 느껴집니다.
- 하단 콜아웃으로 발표자의 해석을 따로 분리한 점도 좋습니다.

### 문제
- 트윗 캡처가 크긴 하지만 텍스트 밀도가 높아 실제 발표 화면에서는 읽기 부담이 큽니다.
- 기존 요청이었던 카파시 이미지 추가는 아직 반영되지 않아 인물의 존재감이 약합니다.
- 왼쪽 bullet과 하단 콜아웃이 비슷한 톤이라, 핵심 문장이 한 번 더 튀지 않습니다.

### 개선 제안
- 트윗은 핵심 문장만 확대해서 보여주고, 원본 전체는 보조 자료로 축소하는 편이 좋습니다.
- 인물 사진이나 프로필을 별도 장면으로 추가해 “누가 말했는가”를 분명히 하세요.
- 왼쪽 카드에서는 bullet보다 결론 문장을 더 크게 두어 첫 계기의 힘을 강조하세요.

## 슬라이드 05
### 좋은 점
- 한국어 번역과 해석 카드가 붙어 있어 메시지가 더 직관적으로 읽힙니다.
- `Coder → Director` 같은 역할 전환 메트릭이 핵심 변화를 잘 압축합니다.
- 04페이지보다 청중이 얻어 가야 할 문장이 더 명확합니다.

### 문제
- 왼쪽 트윗의 텍스트가 너무 많아, 발표 거리에서는 정보가 뭉개질 가능성이 큽니다.
- 오른쪽 카드 3개가 비슷한 박스로 보여 중요도 차이가 충분히 드러나지 않습니다.
- 영어 라벨 `keyword`, `shift`가 한글 중심 슬라이드의 흐름을 살짝 끊습니다.

### 개선 제안
- 트윗 이미지는 핵심 부분만 남기고, 나머지는 과감히 줄여 가독성을 확보하세요.
- 상단 해석 카드의 비중을 더 키우고 하단 메트릭은 보조 레벨로 낮추면 위계가 좋아집니다.
- 라벨도 가능하면 한글로 맞춰 전체 톤을 일치시키는 편이 좋습니다.

## 슬라이드 06
### 좋은 점
- 발표자가 먼저 실험한 사람이라는 점을 사실적으로 증명하는 페이지입니다.
- 좌측과 우측 카드가 사건과 의미를 나눠 담아 구조는 분명합니다.
- 텍스트 중심이지만 비교적 정돈되어 있어 흐름 전환용으로는 무난합니다.

### 문제
- 두 카드의 무게가 비슷해서 어디서 먼저 읽어야 하는지 시선 우선순위가 약합니다.
- 전부 텍스트라 시각적 증거가 부족하고, 화면이 평평하게 느껴집니다.
- 문장형 설명이 많아 발표 리듬이 느려집니다.

### 개선 제안
- 실제 강의나 세미나 사진, 포스터, 캡처 중 하나만 넣어도 신뢰도가 크게 올라갑니다.
- 왼쪽은 사건 타임라인, 오른쪽은 의미 해석으로 더 뚜렷하게 나누세요.
- 핵심 결론 한 줄을 크게 뽑고 나머지는 줄이면 전달력이 좋아집니다.

## 슬라이드 07
### 좋은 점
- `2일`, `6시간`, `MCP`처럼 기억하기 쉬운 숫자와 키워드가 많아 구두 설명에 잘 붙습니다.

- 하단 인사이트 카드가 상단 숫자를 해석으로 연결해 주는 점은 좋습니다.

### 문제

### 개선 제안

- 하단 인사이트는 2~3개 bullet로 바꿔 핵심 교훈을 더 빠르게 읽히게 하세요.
- 결과물 썸네일이나 작은 증거 이미지를 넣으면 제목과 본문의 연결이 더 강해집니다.

## 슬라이드 08
